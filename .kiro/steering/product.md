# 产品概述

## LingFrame (灵珑)

LingFrame 是基于 JVM 的新一代微内核插件化框架，旨在为现代 Java 应用构建**可控、可扩展、可演进**的运行时体系。

### 愿景

让 JVM 应用具备如同操作系统般的插件模型和可控能力。

### 解决的核心问题

- **扩展能力缺乏边界**：扩展逻辑与系统内核解耦，支持独立演进
- **安全仲裁**：通过零信任架构防止业务模块直接访问基础设施（数据库、线程池等）
- **动态加载**：支持可靠的在线热插拔和无缝升级

### 目标用户

- **可扩展产品**：核心产品 + 插件生态模式（IDE、CMS、低代码平台）
- **二次开发平台**：提供标准内核，允许第三方扩展功能
- **SaaS 多租户定制**：不同租户启用不同功能模块
- **插件市场**：用户可自行安装、卸载、升级功能模块
- **大型系统模块化改造**：将单体应用拆分为可独立演进的插件

### 当前状态

🟡 **核心框架已实现** - 插件管理、热重载、权限治理、Spring Boot 3 集成等核心功能已可用。部分基础设施插件和测试仍在完善中。

### 三层架构

```
┌─────────────────────────────────────────────────────────┐
│                    Core（仲裁核心）                      │
│         生命周期管理 · 权限治理 · 能力调度 · 上下文隔离    │
└────────────────────────────┬────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────┐
│           Infrastructure Plugins（基础设施层）           │
│              存储 · 缓存 · 消息 · 搜索                   │
└────────────────────────────┬────────────────────────────┘
                             ▼
┌─────────────────────────────────────────────────────────┐
│             Business Plugins（业务层）                   │
│              用户中心 · 订单服务 · 支付模块               │
└─────────────────────────────────────────────────────────┘
```

### 已实现的核心能力

- **插件生命周期**：安装、卸载、热重载，支持蓝绿部署
- **类隔离**：Child-First 类加载器 + 白名单委派机制
- **Spring 上下文隔离**：每个插件独立的父子 ApplicationContext
- **权限治理**：智能权限推导 + `@RequiresPermission` 显式声明
- **服务扩展**：`@LingService` 注解实现 FQSID 路由的跨插件调用
- **审计追踪**：`@Auditable` 注解 + 异步审计日志
- **开发模式**：文件监听 + 防抖热重载
